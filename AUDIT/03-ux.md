## Proposed New Navigation Tree

### Admin View

```
/admin  (authenticated — shared password gate)
│
├── STICKY CONTROL BAR (persists across all admin pages)
│   ├── Path: Commercial | RRF | Grant | TEPIX Loan | Optima
│   ├── Scenario: Realistic | Upside | Downside | Break-Even
│   ├── Exit: Year · Multiple · €/m²
│   └── Rate/Loan popover (path-conditional)
│
├── ANALYSE
│   ├── Dashboard          (/admin/dashboard)
│   │     KPI grid, 3-scenario return table, DSCR chart + payment-capacity chart,
│   │     DSRA chart (conditional), exit analysis card, conservatism check,
│   │     founder waterfall card, live track record
│   ├── Returns            (/admin/returns)
│   │     Hotel-sale vs property-sale IRR, two-exit table, yield table, MOIC
│   ├── P&L Timeline       (/admin/pnl)
│   │     Year-by-year P&L table with expand/collapse rows
│   ├── Debt Coverage      (/admin/debt-coverage)
│   │     DSCR / LLCR / PLCR detail, DSRA tile, covenant headroom
│   └── Sensitivity        (/admin/sensitivity)   ← absorbs Break-Even
│         ADR sensitivity · occupancy sensitivity · interest sensitivity ·
│         WC sensitivity · CAPEX sensitivity · Break-even matrix (new sub-tab)
│         [Break-even is a single-variable sensitivity; merge, do not keep as separate nav item]
│
├── STRUCTURE
│   ├── Financing Paths    (/admin/financing)   ← moved from Analyse
│   │     Path comparison table (Commercial / RRF / Grant / TEPIX / Optima)
│   │     [Financing is an input-level choice, not an output to analyse;
│   │      but the comparison table is read-only output — place as first item
│   │      in Structure, bridging Inputs and Analyse]
│   ├── OpCo / PropCo      (/admin/opco-split)
│   │     Waterfall mechanics, fee streams, PropCo EBITDA
│   └── Cap Table          (/admin/cap-table)
│         Stakeholder cap table, founder waterfall, per-stakeholder IRR / MOIC / payback,
│         DOCX export
│
└── INPUTS
    ├── Scenarios          (/admin/scenarios)   ← promoted to top
    │     Save / load / share / delete / rename; reference scenario designation
    ├── Assumptions        (/admin/assumptions)
    │   Tabbed sub-editor (9 tabs — see below):
    │   ├── Portfolio         (project count, template assignment, shared costs)
    │   ├── Templates         (per-property CAPEX + OPEX cards)
    │   ├── Financing Paths   (all 5 path param sets — Commercial, RRF, Grant, TEPIX, Optima)
    │   ├── Revenue           (ADR, nights, ramp schedule)
    │   ├── OPEX              (FF&E rate, contingency)
    │   ├── Portfolio OPEX    (staff roles, shared services, overheads, allocation)
    │   ├── General           (tax, working capital, DSRA, OTA distribution)
    │   └── [Remove standalone CAPEX nav item — CAPEX is a sub-section of Templates tab]
    │         [Optima tab stays inside Assumptions, not top-level nav — it's a path config]
    ├── CAPEX Detail       (/admin/capex)   ← keep as nav item for direct access
    │     Per-property CAPEX breakdown; pool config; extra CAPEX lines
    ├── Presentation       (/admin/presentation)
    │     10-section live-wired investor/bank deck; DOCX export
    │     [Currently absent from sidebar; surfaced via Quick Access card on Dashboard only]
    ├── Lexicon            (/admin/lexicon)
    ├── Team               (/admin/team)
    └── Connections        (/admin/connections)
        [Lexicon / Team / Connections are utility items; group visually but keep flat]
```

**Changes from current:**
- Break-Even merged into Sensitivity as a sub-tab (eliminates one nav item)
- Financing Paths moved from Analyse to Structure (it is a structural comparison, not a time-series analysis)
- Scenarios promoted to top of Inputs (it is the most-used item in Inputs)
- Presentation added to Inputs sidebar (currently only reachable via Dashboard Quick Access card)
- Assumptions internal tabs: drop redundant "Optima" top-level tab (it belongs under Financing Paths tab inside Assumptions, not as its own nav item); consolidate DSRA/WC/Tax into a "General" tab

### Bank View

```
/bank  (unauthenticated — name-prompt gate only)
│
├── STICKY BANK CONTROL BAR (BankControlBar)
│   ├── Path pills: Commercial | RRF | Grant | TEPIX | Optima
│   ├── [Optima path → OptimaRatePopover appears inline]
│   ├── Scenario pills: Realistic | Upside | Downside | Break-Even
│   └── Language toggle
│
├── TAB BAR
│   ├── Overview       (current /bank#overview)
│   ├── Credit Analysis  ← rename of current "Sensitivity" tab
│   │     [ADR / occupancy / interest sliders + DSCR/IRR/MOIC outputs]
│   ├── VAT Cashflow   ← promote from inline section in Overview to own tab
│   │     [ConstructionVatCashflow component — currently buried at end of Overview]
│   └── Optima         ← promote /bank/optima as a 4th tab, not a separate URL
│         [Optima is an alternative view of the same deal, not a different site;
│          bankers should not need a separate URL — path pill + tab is sufficient]
│
└── OVERVIEW TAB (restructured section order)
    1. Hero + Quick Access (Tour · Downloads — currently "Coming Soon")
    2. Term Sheet strip (8 KPI cells + Credit Facilities Requested panel)
    3. Deal Overview / Portfolio Table (project summary)
    4. Operating Track Record (LiveTrackRecord)
    5. Capital Structure + Stabilised Ops (2-column)
    6. Loan Metrics (Total Investment / Loan / LTV)
    7. Coverage Ratios (ICR / LLCR / PLCR cards)
    8. CAPEX Breakdown + Sources & Uses
    9. DSCR Trajectory chart + Payment Capacity chart
    10. DSRA (conditional)
    11. Revenue & EBITDA chart
    12. DSCR Scenario Summary table
    13. Financing Path Comparison table
    14. All-Paths DSCR Trajectory chart
    15. P&L Timeline (BankPnLSection)
    → STRESS ANALYSIS moved to Credit Analysis tab (currently at end of Overview)
```

**Changes from current:**
- "Sensitivity" tab renamed to "Credit Analysis" (more meaningful to a credit analyst)
- VAT Cashflow promoted to own tab (currently last item in Overview, easy to miss)
- Stress Analysis (collateral + cash-flow) moved out of Overview into Credit Analysis tab (it belongs with the sensitivity tools)
- Optima view promoted to 4th tab on /bank, eliminating the separate /bank/optima URL; BankControlBar already handles path switching — the dedicated URL creates unnecessary navigation ambiguity
- P&L moved above Stress Analysis in Overview (evidence before verdict)

## Task Flow Analysis

### Admin Flows

**Flow A: Change a key assumption and see DSCR impact**
1. From any page → sidebar → Assumptions (Inputs group, item 2 after CAPEX) — 2 clicks
2. Assumptions → Financing tab (currently tab 4, after Portfolio/Templates/Optima) — 1 tab click
3. Change value — model recalculates live
4. Navigate to Dashboard to see DSCR chart — 2 clicks (sidebar Analyse → Dashboard)
**Total: 5 clicks + scrolling past 3 assumptions tabs**
After UX-05 fix: Financing becomes tab 2 → 1 fewer tab click. After UX-07: in-page anchor bar skips scrolling. Practical path becomes 4 clicks.

**Flow B: Compare three financing paths**
1. Sidebar → Financing Paths (Analyse, last item) — 2 clicks
2. Path comparison table loads — done
**Total: 2 clicks. Already good.** After UX-02, Financing Paths moves to Structure group — still 2 clicks but better IA placement.

**Flow C: Save a scenario and share it**
1. Sidebar → Scenarios (Inputs, item 3) — 2 clicks
2. Click Save → name modal → save — 2 actions
3. Firestore publishes; copy share link — 1 action
**Total: 2 clicks + 3 actions. Acceptable.** After UX-04 (Scenarios promoted to first in Inputs): 2 clicks to reach, not 3.

**Flow D: Export the bank presentation**
1. Dashboard → Quick Access card → Presentation → new tab opens `/presentation` (PDF viewer) — 3 clicks
   OR: `/admin/presentation` for live-wired DOCX — requires knowing the URL (not in sidebar)
**Worst offender: `/admin/presentation` is not findable in 3 clicks from /admin/sensitivity or /admin/pnl.** After UX-03 fix: sidebar → Presentation = 2 clicks from anywhere.

### Bank Flows

**Flow A: Review deal overview and key credit metrics**
1. Land on /bank → scroll past hero and Quick Access → reach Term Sheet (section 3) — 1 scroll
2. Term Sheet shows: loan, term, rate, DS, DSCR, equity, total, security — all critical metrics present
**No structural problem; term sheet is near the top.** UX-25 (in-page anchor bar) would make it zero-scroll.

**Flow B: Run a stress test**
1. /bank → Overview tab is default → scroll to section 12 (Stress Analysis) — 14 sections of scroll
**Worst offender in bank view.** After UX-14: Stress moves to Credit Analysis tab = 1 tab click from landing.

**Flow C: Review construction VAT cashflow**
1. /bank → Overview tab → scroll past 14 sections to ConstructionVatCashflow — very long scroll
**Second worst offender.** After UX-13: VAT Cashflow tab = 1 tab click.

**Flow D: Compare Optima-specific terms**
1. A banker with only the /bank URL has no path to /bank/optima — zero-click dead end
2. An admin must send /bank/optima separately
**Complete findability failure.** After UX-12/UX-22: Optima tab on /bank = 1 tab click.
